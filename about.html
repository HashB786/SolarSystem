About This Project

A deep dive into how Space Explorer was designed and built — from code architecture to the design decisions that bring the cosmos to life in your browser.

🚀 The Vision

Space Explorer was built to make astronomy accessible and exciting. The goal: create a visually immersive, interactive experience that feels like stepping into mission control — all without any frameworks, build tools, or server-side dependencies beyond a CDN.

Every design decision was intentional. The dark space theme reduces eye strain for extended sessions, the animated starfield creates depth without distraction, and the neon glow effects evoke science fiction aesthetics while remaining functional and accessible.

How It Was Built

H5

Semantic HTML5

Every page uses proper semantic elements: <nav>, <main>, <section>, <article>, <header>, and <footer>. ARIA attributes ensure screen-reader compatibility, and all interactive elements have descriptive labels.

TW

Tailwind CSS

Tailwind's utility-first classes handle the responsive grid, spacing, typography, and hover transitions. Loaded via CDN with a custom config that extends the theme with space-themed colours and fonts. Mobile-first using sm:, md:, lg: breakpoints throughout.

JS

Vanilla JavaScript ES6+

All interactivity lives in a single script.js. Written with arrow functions, template literals, destructuring, and Array.filter(). Handles: hamburger menu, planet modal, quiz engine, dark mode, scroll-to-top. Zero dependencies — just open in a browser.

🎨

Canvas API + localStorage

The animated starfield uses the HTML5 Canvas API — 220 stars with individual twinkle speeds, opacity oscillation via Math.sin(), and slow drift via requestAnimationFrame(). localStorage persists dark-mode preference and quiz high scores across sessions.

Technologies Used

HTML5 Tailwind CSS JavaScript ES6+ Canvas API Google Fonts localStorage CSS Animations CSS Custom Properties

🎓 What I Learned

  • ✦
    Canvas API & requestAnimationFrame()

    Creating smooth 60fps animations without any library — managing canvas size on resize and using Math.sin() to simulate natural twinkling motion.

  • ✦
    CSS Glassmorphism

    Combining backdrop-filter: blur(), semi-transparent backgrounds, and subtle borders to create the "frosted glass" card effect that defines modern dark-theme UIs.

  • ✦
    Mobile-first responsive layout with Tailwind

    Tailwind's breakpoint prefixes force you to think small-first. The pattern grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 is now instinctive for building adaptive layouts.

  • ✦
    State management without a framework

    Managing quiz state (current question, score, answered flag) with a plain JavaScript object taught me the fundamentals of UI state that frameworks like React abstract away.

  • ✦
    CSS Radial Gradients as planet graphics

    You don't always need images. Each planet is a <div> with layered radial-gradient() and repeating-linear-gradient() rules creating a realistic 3D sphere illusion.

  • ✦
    localStorage for cross-session persistence

    localStorage.getItem() and setItem() let user preferences like dark mode and quiz high scores survive page loads and browser restarts — no server required.

🌌

Built with Curiosity

This project was an exploration — not just of the solar system, but of what's achievable with modern web standards and zero build tools. Pure HTML, CSS, and JavaScript can still build something beautiful.
